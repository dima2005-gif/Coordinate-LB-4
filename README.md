- Чому ```learning_rate``` є найважливішим гіперпараметром градієнтного спуску?

- занадто малий -> стагнація та дуже повільна збіжність

Алгоритм у 3c все одно сходиться, оскільки він мінімізує функцію втрат відносно наданих (зашумлених) даних, тому графік Loss зменшується.

Однак через дуже високий рівень шуму вхідні дані є сильно спотвореними, тому алгоритм знаходить мінімум, який відповідає саме цим неправильним даним, а не реальному положенню.

Тобто він збігається до хибного (локального) мінімуму, що відповідає зашумленим вимірюванням, через що фінальна похибка є дуже великою
